## 3. 데이터베이스 연결 설정

* app/core/db/databases.py 관련 내용:
  * SQLAlchemy 연동 소스코드를 바탕으로 데이터베이스와 애플리케이션을 연결하는 설정이 작성됩니다.
  * 1. create_engine: 설정된 데이터베이스 URL 주소를 기반으로 실제 데이터베이스와의 물리적 연결 통로를 생성합니다.
  * 2. sessionmaker: 데이터베이스 연산(조회, 저장 등)을 실행하는 창구인 세션(Session) 객체를 생성해 주는 공장 역할을 수행합니다.
  * 3. get_db(): FastAPI의 Depends(의존성 주입)를 활용하여 API 요청이 들어올 때 세션을 생성하고(yield), API 처리가 완전히 끝나면 에러가 나더라도 finally 구문을 통해 안전하게 세션을 닫아주는(db.close()) 역할을 수행합니다.

## 5. API 구현 흐름

  FastAPI 기본 동작, Pydantic 데이터 검증, 그리고 SQLAlchemy 데이터베이스 연동까지 포함된 실행 흐름입니다.

1. [클라이언트의 API 요청 진입]
   * 사용자가 브라우저나 도구를 통해 특정 주소로 요청을 보냅니다. (데이터 조회는 GET, 데이터 등록은 POST)
2. app/main.py (FastAPI 앱 구동)
   * Uvicorn 서버를 통해 API 요청을 가장 먼저 받아들입니다.
3. app/apis/ (Router 및 HTTP 메서드 매칭)
   * @app.get 또는 @app.post 처럼 요청 종류와 주소에 맞는 핸들러 함수를 찾아 연결합니다.
4. app/schemas/ (Pydantic 데이터 검증)
   * 요청을 통해 들어온 데이터가 있다면 BaseModel을 활용해 데이터 규격이 올바르게 들어왔는지 입구에서 검사합니다.
5. app/core/db/databases.py (데이터베이스 세션 주입)
   * get_db() 의존성 주입을 통해 해당 요청을 처리할 안전한 데이터베이스 세션을 획득합니다.
6. app/models/ (SQLAlchemy ORM 연산)
   * 주입받은 데이터베이스 세션과 정의된 ORM 모델을 활용하여 데이터베이스에 데이터를 저장하거나 필요한 데이터를 조회합니다. 
7. [클라이언트에게 최종 응답 완료]
   * 처리가 완료된 결과나 조회된 데이터를 JSON 형태로 클라이언트에게 다시 반환하고 세션을 안전하게 닫습니다.
