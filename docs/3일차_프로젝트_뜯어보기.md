# 3일차 프로젝트 뜯어보기

## 1. 디렉터리 역할 및 작성 내용

### `app/core/`
프로젝트 전역 설정 및 공통 인프라를 담당합니다.

| 파일 | 역할 |
|------|------|
| `config.py` | 환경변수(.env) 로드 및 전역 설정 관리 |
| `db/databases.py` | SQLAlchemy 엔진, 세션 생성 |
| `db/models.py` | 모든 ORM 모델의 Base 클래스 정의 |

---

### `app/models/`
SQLAlchemy ORM 모델(테이블 정의)을 작성하는 디렉터리입니다.  
각 테이블마다 별도의 `.py` 파일을 생성합니다.

```
app/models/
├── user.py          # users 테이블
├── patient.py       # patients 테이블
├── medical_record.py # medical_records 테이블
├── xray_image.py    # xray_images 테이블
└── ai_analysis.py   # ai_analysis_results 테이블
```

---

### `app/repositories/`
데이터베이스 CRUD 로직을 담당합니다.  
Service 계층과 DB 사이의 중간 계층으로, 직접 쿼리를 실행합니다.

```
app/repositories/
├── user_repository.py
├── patient_repository.py
└── ...
```

예시:
```python
class UserRepository:
    def get_by_id(self, db: Session, user_id: int) -> User: ...
    def create(self, db: Session, data: dict) -> User: ...
```

---

### `app/schemas/`
Pydantic 모델을 작성하는 디렉터리입니다.  
API 요청(Request) / 응답(Response) 데이터의 유효성 검사 및 직렬화를 담당합니다.

```
app/schemas/
├── user.py       # UserCreate, UserResponse 등
├── patient.py
└── ...
```

---

### `app/services/`
비즈니스 로직을 담당합니다.  
Repository를 호출하여 데이터를 처리하고 결과를 반환합니다.

```
app/services/
├── user_service.py
├── patient_service.py
└── ...
```

---

### `app/apis/`
FastAPI 라우터(엔드포인트)를 작성하는 디렉터리입니다.  
HTTP 요청을 받아 Service를 호출하고 응답을 반환합니다.

```
app/apis/
├── user_router.py
├── patient_router.py
└── ...
```

---

## 2. 주요 파일 역할

### `app/main.py`
FastAPI 애플리케이션의 진입점입니다.  
앱 인스턴스 생성, 라우터 등록, 미들웨어 설정 등을 담당합니다.

```python
from fastapi import FastAPI
from app.apis import user_router, patient_router

app = FastAPI()
app.include_router(user_router.router)
app.include_router(patient_router.router)
```

---

### `app/core/config.py`
`.env` 파일의 환경변수를 읽어 전역 설정으로 제공합니다.  
`pydantic-settings`의 `BaseSettings`를 활용합니다.

```python
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    DATABASE_URL: str

    class Config:
        env_file = ".env"

settings = Settings()
```

---

### `pyproject.toml`
프로젝트의 메타데이터와 의존성 패키지를 정의하는 파일입니다.  
`uv`를 사용할 경우 `pip install` 대신 이 파일 기준으로 패키지를 설치합니다.

---

### `uv.lock`
`uv`가 생성하는 의존성 잠금 파일입니다.  
팀원 모두가 동일한 패키지 버전을 사용할 수 있도록 보장합니다. (직접 수정 금지)

## 3. 데이터베이스 연결 설정 (`app/core/db/databases.py`)

SQLAlchemy를 사용하여 DB 엔진과 세션을 설정합니다.

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
from app.core.config import settings

engine = create_engine(settings.DATABASE_URL)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

## 4. SQLAlchemy 모델 작성 및 Alembic 마이그레이션

### SQLAlchemy 모델 작성 예시

```python
# app/models/user.py
from sqlalchemy import Column, Integer, String, Boolean, DateTime, Enum
from app/core/db/models import Base
import enum

class RoleEnum(enum.Enum):
    PENDING = "PENDING"
    STAFF = "STAFF"
    ADMIN = "ADMIN"

class User(Base):
    __tablename__ = "users"

    id = Column(Integer, primary_key=True, autoincrement=True)
    email = Column(String(255), unique=True, nullable=False)
    hashed_password = Column(String(255), nullable=False)
    name = Column(String(20), nullable=False)
    role = Column(Enum(RoleEnum), nullable=False)
```

### Alembic 마이그레이션 수행 방법

```bash
# 1. alembic 초기화 (최초 1회)
alembic init alembic

# 2. alembic.ini에서 DB URL 설정
# sqlalchemy.url = postgresql://user:pass@localhost/dbname

# 3. 마이그레이션 파일 자동 생성
alembic revision --autogenerate -m "create initial tables"

# 4. DB에 적용
alembic upgrade head

# 5. 롤백
alembic downgrade -1
```

## 5. API 구현 흐름

```
HTTP 요청
   ↓
[apis/] 라우터  →  요청 데이터 수신 (Pydantic Schema로 검증)
   ↓
[services/] 서비스  →  비즈니스 로직 처리
   ↓
[repositories/] 레포지토리  →  DB CRUD 실행
   ↓
[models/] ORM 모델  →  실제 DB 테이블과 매핑
   ↓
HTTP 응답  →  Pydantic Schema로 직렬화하여 반환
```

각 계층이 분리되어 있어 유지보수성과 테스트 용이성이 높아집니다.
