## 팀 정보

| 항목 | 내용 |
|------|------|
| 팀명 | 3조 |
| 팀원 | 3명 |
| 프로젝트 진행 시간 | 15:30 ~ 16:30 |

---

## 팀 규칙

### 1. 커뮤니케이션
- 모르는 것이 있으면 팀원에게 적극적으로 질문한다.
- 작업 시작 전 오늘 할 일을 팀원과 공유한다.
- 작업 완료 후 진행 상황을 팀원에게 알린다.
- 중요한 결정은 팀원 전원이 동의한 후 진행한다.

### 2. Git & Github 규칙
- `main` 브랜치에 직접 push 하지 않는다.
- 작업 시작 전 반드시 `main` 브랜치를 pull 받는다.
- 브랜치 네이밍 규칙을 지킨다: `feature/기능명`
- PR 생성 시 팀원 1명 이상의 리뷰를 받은 후 merge 한다.

### 3. 커밋 메시지 규칙
```
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 수정
style: 코드 포맷 변경
refactor: 코드 리팩토링
chore: 기타 변경사항
```

### 4. 코드 작성 규칙
- 변수명, 함수명은 의미있게 작성한다.
- 코드 작성 후 주석을 달아 이해하기 쉽게 한다.
- 작동하지 않는 코드는 push 하지 않는다.

### 5. 회의 규칙
- 프로젝트 진행 시간(15:30 ~ 16:30)에 적극적으로 참여한다.
- 회의 시간을 준수한다.
- 결정된 사항은 모두 기록으로 남긴다.

### 6. 갈등 해결
- 의견 충돌 시 서로 존중하며 대화로 해결한다.
- 해결이 어려운 경우 팀 투표로 결정한다.
